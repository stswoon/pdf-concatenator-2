# AGENTS.md 

Инструкции для AI-агентов, работающих с этим репозиторием.

## О проекте

Минимальное SPA на **React 19 + TypeScript + Vite 6**. Стартовая точка для экспериментов с AI-инструментами и фронтенд-разработкой. Сейчас — экран «Hello World» без роутинга, стейт-менеджмента и API.

## Стек

| Слой | Технология |
|------|------------|
| UI | React 19, функциональные компоненты |
| Язык | TypeScript 5.8 (strict) |
| Сборка | Vite 6, `@vitejs/plugin-react` |
| Стили | Обычный CSS (`src/index.css`) |

## Структура

```
test-ai/
├── index.html          # Точка входа HTML, подключает /src/main.tsx
├── src/
│   ├── main.tsx        # createRoot, StrictMode, глобальные стили
│   ├── App.tsx         # Корневой компонент приложения
│   ├── index.css       # Глобальные стили
│   └── vite-env.d.ts   # Типы Vite
├── vite.config.ts
├── tsconfig.json       # Project references
├── tsconfig.app.json   # Настройки для src/
└── tsconfig.node.json  # Настройки для vite.config.ts
```

Не редактировать: `dist/`, `node_modules/`, `.idea/`.

## Команды

```bash
npm install      # установка зависимостей
npm run dev      # dev-сервер (Vite, HMR)
npm run build    # tsc -b && vite build → dist/
npm run preview  # локальный просмотр production-сборки
```

Перед завершением задачи с изменениями кода запускай `npm run build` и исправляй ошибки TypeScript и сборки.

## Соглашения по коду

### TypeScript

- `strict: true`, `noUnusedLocals`, `noUnusedParameters` — не оставляй неиспользуемый код.
- `verbatimModuleSyntax: true` — используй `import type` для импорта только типов.
- Новый код размещай в `src/`.

### React

- Только функциональные компоненты и хуки.
- Корневой компонент — `App.tsx`; точка монтирования — `main.tsx`.
- Оборачивай приложение в `StrictMode` (как в `main.tsx`).

### Стиль кода (как в существующих файлах)

- Одинарные кавычки в импортах.
- Без точек с запятой в конце строк.
- Именованные импорты из React: `import { StrictMode } from 'react'`.
- Стили — через `className` и CSS-классы; глобальные стили в `index.css`, при росте проекта — colocate CSS рядом с компонентом.

### Пример компонента

```tsx
function MyComponent() {
  return (
    <section className="my-component">
      <h2>Title</h2>
    </section>
  )
}

export default MyComponent
```

## Правила для агента

### Делай

- Минимальный diff: меняй только то, что нужно для задачи.
- Следуй существующим паттернам; перед добавлением библиотек проверь `package.json`.
- Сохраняй простую структуру, пока проект маленький — не вводи лишние слои (FSD, DI и т.п.) без явной просьбы.
- Коммиты создавай только по явной просьбе пользователя.

### Не делай

- Не коммить `.env`, ключи API и секреты.
- Не править `dist/` и артефакты сборки.
- Не добавлять тесты, README и документацию, если пользователь не просил.
- Не раздувать scope: без запроса не добавляй роутер, UI-библиотеки, ESLint/Prettier, Tailwind и т.д.

## Добавление зависимостей

1. Установи через `npm install <package>`.
2. Обнови импорты и конфиг только при необходимости.
3. Проверь `npm run build`.
