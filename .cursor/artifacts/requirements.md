# Requirements: PDF Concatenator

## Цель

Единое client-side SPA для работы с PDF и изображениями: импорт файлов, предпросмотр страниц в общей рабочей области, изменение порядка, сжатие изображений, экспорт в PDF или отдельные PNG/JPG. Вся обработка — в браузере, без отправки файлов на сервер.

## In scope

### Импорт и рабочая область

- Приём файлов через drag & drop и `<input type="file">` (multiple).
- Поддерживаемые форматы: **PDF**, **JPG/JPEG**, **PNG**.
- PDF при импорте раскладывается на отдельные страницы в общем списке.
- Каждая JPG/PNG добавляется как одна страница.
- Превью каждой страницы (thumbnail) в сетке/ленте.
- Удаление отдельной страницы из рабочей области.
- Drag & drop изменение порядка страниц.

### Операции

| # | Операция | Описание |
|---|----------|----------|
| 1 | Split → PNG/JPG | Экспорт выбранных или всех страниц как отдельные изображения (ZIP при >1 файле). PDF-страницы рендерятся через pdf.js; image-страницы отдают текущие байты. |
| 2 | Merge PDF | Склеивание нескольких PDF (через общую рабочую область или импорт нескольких PDF подряд). |
| 3 | Images → PDF | Склеивание JPG/PNG в PDF через общую рабочую область + Export PDF. |
| 4 | Compress | Для image-страниц: настройка качества (JPEG) и/или максимальной стороны; preview размера; скачать сжатые файлы или использовать при Export PDF. |
| 5 | Reorder | Drag & drop страниц (см. sunny-day). |
| 6 | Text-safe export | PDF-страницы из исходного документа экспортируются через `copyPages` без растеризации; image-страницы встраиваются как изображения на A4. |

### Правила размещения изображений на A4

- Размер «холста» A4: **2480 × 3508 px** (300 DPI).
- Если изображение **больше** A4 — масштабировать **пропорционально**, чтобы целиком вписаться; позиция **верхний левый угол** (0, 0).
- Если изображение **меньше или равно** A4 — **не растягивать**; разместить в верхнем левом углу в исходном (или уже сжатом) размере.
- PDF-страница A4 создаётся через pdf-lib с размерами, соответствующими 2480×3508 pt-scale (595.28×841.89 pt — стандарт A4).

### UX минимум

- Одна главная рабочая зона (без роутера).
- Панель действий: Add files, Export PDF, Export images (PNG/JPG), Compress (для image-страниц).
- Индикатор загрузки/обработки при тяжёлых операциях.
- Сообщения об ошибках (неподдерживаемый формат, повреждённый файл, зашифрованный PDF).

## Out of scope

- Серверная обработка, аккаунты, облачное хранилище.
- Редактирование текста PDF, аннотации, формы, подписи, водяные знаки.
- OCR, распознавание текста.
- Поворот страниц, обрезка, нумерация (если не запросят отдельно).
- Работа с зашифрованными PDF (password-protected) — только сообщение об ошибке.
- Пакетная автоматизация, CLI, API.
- Мобильное нативное приложение.
- i18n (достаточно UI на английском или русском — на усмотрение Dev, один язык).
- Автотесты и CI (только по явной просьбе).

## Ограничения

- **Стек:** React 19, TypeScript strict, Vite 6, Tailwind, shadcn, Zustand — без новых архитектурных слоёв (FSD, роутер).
- **Новые зависимости:** допустимы только PDF-специфичные — `pdf-lib`, `pdfjs-dist` (+ типы при необходимости). Drag & drop — нативный HTML5 / pointer, без `@dnd-kit` без запроса.
- **Память:** лимит брактичен RAM браузера; рекомендуемый soft-limit — до ~100 страниц / ~200 MB суммарно с предупреждением.
- **Приватность:** файлы не покидают клиент.
- **Сборка:** `npm run build` должен проходить без ошибок.

## Sunny-day сценарии заказчика

### SD-1: PDF + PNG → merged PDF с сохранением текста

1. Добавить PDF из 2 текстовых страниц → отображаются 2 thumbnail.
2. Добавить PNG → отображается 3-я страница.
3. PNG вписан в A4 (2480×3508), top-left, без upscaling.
4. Перетащить PNG между 1-й и 2-й PDF-страницами.
5. Нажать **Export PDF**.
6. Скачивается один PDF из 3 страниц в порядке: PDF-1, PNG, PDF-2.
7. Размер файла ≈ размер исходного PDF + вклад PNG; текст остаётся selectable/searchable в PDF-viewer.

### SD-2: Split PDF в PNG

1. Добавить PDF из N страниц.
2. Export images → PNG.
3. Скачивается ZIP с N файлами `page-001.png` … `page-00N.png`.

### SD-3: Compress + merge images

1. Добавить 3 JPG.
2. Уменьшить quality до 70%, скачать или оставить в workspace.
3. Export PDF → один PDF из 3 страниц, каждая — A4 с image top-left.

## Открытые вопросы

**Критичных открытых вопросов нет.** Принятые допущения (некритичные):

- UI язык: русский или английский — на усмотрение Dev (один язык).
- Export нескольких image-страниц: ZIP-архив (стандарт для ilovepdf-like UX).
- Soft-limit памяти: предупреждение, не hard block.
